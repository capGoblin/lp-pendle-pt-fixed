## 1. What the Hackathon Actually Is

A **3-week** online hackathon hosted on DoraHacks. Three orgs co-run it:
- **BNB Chain** (chain + AI Agent SDK)
- **CoinMarketCap** (data, MCP server, Skills Marketplace, x402)
- **Trust Wallet** (Agent Kit — self-custody signing)

The framing is explicit: "Build the Autonomous Trading Agent Stack." Not "build something Web3." Specifically: **AI agents that read market signals, decide, and execute on BSC**.

Two tracks + 3 sponsor specials. **$36K pool.**

### Track 1 — Autonomous Trading Agents
- **Prize:** $24K, 5 winners (1st $10K / 2nd $6K / 3rd $4K / 4th $2K / 5th $2K)
- **Format:** Ship an agent that **trades live on BSC** during the **Jun 22–28 trading window**
- **Scoring:** **Real PnL** (subject to max drawdown cap)
- **Recommended stack:** CMC MCP server + Trust Wallet Agent Kit + BNB AI Agent SDK
- **Execution surfaces:** PancakeSwap, BSC perps

### Track 2 — Strategy Skills
- **Prize:** $6K, 3 winners (1st $3K / 2nd $2K / 3rd $1K)
- **Format:** Build a **CMC Skill** — a backtestable strategy spec, no live execution
- **Scoring:** Discretionary expert panel (technical execution, originality, real-world relevance)
- **Optional:** x402 pay-per-call

### Special Prizes ($6K total, $2K each, stackable)
- Best Use of CMC Data & Signal
- Best Use of Trust Wallet Agent Kit
- Best Use of BNB AI Agent SDK

### Non-cash benefits for finalists
- CMC Pro API credits
- Claude API tokens (or equivalent compute)
- 1 mentor per finalist team
- Kickstart Package shortlist eligibility
- TWT Developer Portal listing

**Stack-all-three observation:** The CMC page literally tells you to "stack all three." This is a hint about the winning formula: integrate CMC + TWAK + BNB SDK deeply.

## 2. Why This Matters (Builder Lens)

This is not a generic Web3 hackathon. It's a **vertically-aligned agent infra demo** — the three sponsors are each contributing a critical piece of the agent-trading stack (data, custody, chain). Anyone who can wire all three together cleanly gets disproportionate attention.

For an operator positioning as a top-0.1% builder in Crypto x AI:
- Winning here = compounding credibility + Kickstart eligibility + CMC/TWT/BNB signal
- Top-3 even in Track 2 = CMC mentorship, Pro API credits, listed on TWT dev portal
- Even non-winning submissions get visible in a small competitive set → twitter content + open-source footprint

The kicker: **the submission window closes in ~38 hours.** So this research plan has to converge on a build recommendation within hours, not days.

## 4. Hypotheses to Test (Pre-Research Anchors)

These are priors — to be validated or killed by Phase A/B research:

1. **The hackathon wants a complete end-to-end stack demo, not just one layer.** Stack-all-three is the explicit hint.
2. **Track 1 (live trading) is riskier but has bigger prizes.** Real PnL means real variance — most teams will submit a defensive agent, an aggressive one will likely win.
3. **Skills Marketplace (Track 2) is the sleeper lane.** Lower competition, monetizable post-hackathon via x402 + Skills Marketplace, and you don't have to win the live PnL lottery.
4. **Verifiable AI is the underbuilt wedge.** Every team will trust CMC's data and trust the agent's output. Nobody is going to verify the agent's reasoning with TEE/zkML. That's the operator's adjacent vertical — the highest-signal lane.
5. **Multi-agent is underbuilt.** Single LLM-call agents are the entry bar. Splitting signal / risk / execution into specialized agents with bounded authority is harder but more defensible.
6. **The Kickstart Package is the real prize for serious builders.** Eligibility shortlist matters more than the $10K. So the build needs to be **fundable** post-hackathon, not just demoable.

## 6. Decision Gates

- **Gate 1 (after Phase A):** Is the submission process solo-friendly and is the operator eligible? If KYC demands entity formation or region blocks, kill the plan.
- **Gate 2 (after Phase B):** Is the underbuilt wedge actually buildable in 36h by one person? If not, downgrade to Track 2 or a simpler Track 1 idea.
- **Gate 3 (after Phase C):** Are the SDKs actually ready? If TWAK or BNB SDK are vaporware, the recommended stack collapses and we should pivot.
- **Gate 4 (after Phase D):** Is the chosen idea **fundable** post-hackathon (Kickstart + CMC + TWT follow-on)? If not, downgrade to a portfolio-piece-only submission.
