## 6. Mapped Gaps (whitespace for a 3-week hackathon entry)

### Gap 1: **Real zkML / cryptographic inference proofs on BNB**

The BNB "verifiable AI" lane is **Merkle proofs (Singularry) + BAS attestations**. **No live project is doing cryptographic zkML proofs of inference on BNB.** Only **Brevis** (omnichain ZK coprocessor) supports BNB with ZK proofs. An agent that uses Brevis or Modulus-Lite for on-chain ZK inference proofs would have a strong differentiator against **both** the Track 1 PnL race **and** the TWAK special prize (where "verifiable AI" is currently Singularry's claim of differentiation).

### Gap 2: **Strategy Skill as an MCP server for downstream agents**

Track 2 ($6K, 3 winners) requires a CMC Skill that turns market data into a strategy spec. The spec example is "momentum Skill blending RSI + MACD + Fear & Greed" — clearly meant as a **building block**, not a standalone product. **The winning Track 2 entry is one that's also MCP-exposed so it can be consumed by other Track 1 agents.** This also positions the operator for the **Best Agent Hub Use** special prize.

### Gap 3: **Auditable PnL reputation primitive (BAS Agent Passport integration)**

BAS is launching Agent Passport. No one is building a **trading-PnL reputation primitive** — an on-chain record of an agent's live trading performance, attestable and composable. **The June 22–28 trading window is the perfect proving ground.** Whoever ships an "agent PnL index" by the deadline wins the post-hackathon narrative even if they don't take 1st.

### Gap 4: **TWAK as the SOLE execution layer (max 30 pts)**

The TWAK special prize explicitly rewards TWAK being the **sole execution layer** with multiple surfaces (signing + autonomous mode + x402). Most hackathon entries will use TWAK + something else (PancakeSwap router, custom contracts, etc.). **A pure-TWAK entry that uses TWAK for swap + autonomous mode + x402 data payments** scores the ceiling on integration depth.

### Gap 5: **x402 in the actual trade loop (10 pts on TWAK score)**

The TWAK score includes 10 pts for **"x402 used to pay per request for data, inference, or tools as part of its trade loop"**. This is the lowest-weight criterion but also the most overlooked. An agent that pays for each CMC Hub call via x402 (instead of a flat-rate API key) demonstrates a fully autonomous loop. **TWAK has native x402** so this is a free 10-pt unlock for anyone using TWAK.

### Gap 6: **Self-custody ladder maximum (20–25 pts)**

The brutal penalty is for custody in the core loop. **TWAK's whole point is local signing, so any TWAK-based entry should auto-score 20–25 here.** But: if the entry uses a relayer, a meta-tx pattern, or a custodial co-signer anywhere in the trade path, the score collapses. **Discipline: every signing operation must end at TWAK's local signer. No shortcuts.**

### Gap 7: **Risk-managed perps / structured products agent**

Aster is the perps DEX (ASTER in eligible token list). But there's no agent doing **risk-managed perpetuals or structured products** (funding-rate harvesting, delta-neutral vol selling, options strategies) on BNB. WORLD3 + Bullbit just announced a perp partnership. **A focused strategy agent is the solo-builder wedge before WORLD3/Bullbit generalize.**

### Gap 8: **Composite strategy that uses CMC data TWAK doesn't naturally surface**

CMC Hub covers CEX + derivatives + on-chain + social + KOLs + news. Most hackathon agents will use the obvious bits (price + funding rates). **The wedge is using the less-obvious CMC data — KOL signal, news sentiment, social heat — to drive trades.** The spec literally calls out "an agent that combines CMC funding rates and Fear & Greed with TWAK auto-execution" — that's the *example*, not the ceiling.

## 7. Strategic Recommendation

### The single best play (one project, all prizes)

**Build a TWAK-native trading agent that:**

1. **Reads markets via CoinMarketCap AI Agent Hub** (MCP + CLI + Skills + x402 payments per call)
2. **Reasons via LLM** (any provider; spec doesn't mandate one)
3. **Executes via TWAK as the sole execution layer** (no other DEX/router, no custom contracts, no relayer)
4. **Uses TWAK's native x402 to pay per CMC call** (free 10 pts)
5. **Has verifiable inference** via either (a) Merkle proof commits on BAP-578 NFA, (b) Brevis ZK coprocessor for zkML proofs, or (c) BAS attestation of each decision
6. **Exposes its strategy as an MCP server** so other agents can subscribe to its signals (Track 2 entry path, Agent Hub special prize, post-hackathon product)
7. **Runs one strategy** that's well-tested pre-June-22 (backtested via Track 2 Skill, live tradeable via Track 1)
8. **Has TWAK-native risk guardrails** (drawdown cap, token allowlist of the 149 BEP-20s, per-trade + daily limits, slippage protection)

**Why this works for the scoring:**

- **Track 1 PnL race:** Run a sensible, risk-managed strategy. Don't aim for the highest PnL — aim for the highest risk-adjusted PnL under the 30% drawdown cap. Singularry's regime classifier + RL stack is hard to beat, but Singularry isn't using TWAK, so the PnL bracket where this entry wins is "solidly positive, no blowup."
- **TWAK Special Prize:** Score ceiling or near-ceiling on all six criteria if execution is disciplined. **Estimated 85–95 / 100.**
- **Best Agent Hub Use:** Top scores if the agent genuinely uses CMC Hub across multiple surfaces (CEX + derivatives + on-chain + social + KOLs + news) via MCP + x402 + CLI.
- **Best BNB SDK Use:** If the SDK is used for any agent orchestration layer (memory, discovery via 8004scan, BAS reputation commits).
- **Track 2 fallback:** The strategy logic, abstracted from the execution layer, becomes a standalone CMC Skill.

**Realistic for solo builder in 3 weeks?**
Yes, if scope is disciplined. Two big risks:
- **Risk: Custom ZK proofs take longer than 3 weeks.** Mitigation: ship Merkle-proof commits (faster) for the live trading demo, and ship zkML as a research extension / post-hackathon roadmap.
- **Risk: Strategy underperforms in the live week.** Mitigation: pre-backtest against historical data, ship a "risk-parity basket" strategy that's hard to blow up rather than a moonshot directional bet.

### Specific strategy ideas worth considering

The spec literally hints at three:
1. **"Funding rates + Fear & Greed + auto-execution rotating between BSC perps"** — funding-rate arbitrage. Classic. Tight risk profile, solid expected return. **Use ASTER (perp venue), data via CMC.**
2. **"DCA agent with a personality that is sentiment-aware, talks back, and signs its own txs"** — personality-driven DCA. Lower PnL ceiling but high originality. **Strong demo narrative.**
3. **"Copy-trader that mirrors top wallets through your own risk filters"** — wallet-mirror. Needs on-chain wallet analytics (CMC covers this).

**Other viable strategies:**
4. **Momentum + KOL-signal fusion** — buy when KOL sentiment aligns with momentum, sell when it diverges. Uses less-obvious CMC data (social + KOLs). Originality bonus.
5. **Regime-switching strategy** — bull = trend-follow, bear = mean-revert, sideways = range-trade. Regime detection via derivatives positioning (also CMC data).
6. **Sentiment-divergence strategy** — flag when social heat says "bullish" but on-chain flow says "smart money exiting." Trade the divergence.

### My recommendation for the operator

**Pick one strategy and execute it cleanly.** The strategy choice matters less than the execution quality. The 4 judges (technical / originality / relevance / demo) reward a **clean, demo-able, end-to-end agent** that demonstrably:
- Uses TWAK as the sole execution layer (not a router + TWAK)
- Pays for CMC data via x402 (proves the autonomous loop)
- Has visible guardrails (drawdown cap, allowlist, daily limits)
- Demonstrates verifiable inference (Merkle or BAS)
- Has a memorable demo (one clear trade decision traceable from input → reasoning → tx hash on BSC)

The four-track structure is **not** a suggestion to ship four things. It's a suggestion that **one well-built project can win multiple prizes**.

### Backup wedge: PnL reputation primitive

If the operator can't ship the full agent in 3 weeks, the **second-best play** is a **PnL reputation primitive** that:
- Indexes all hackathon participants' live trading performance
- Mints BAS Agent Passport attestations for each
- Publishes a public leaderboard (on-chain + web UI)
- Documents the strategies as CMC Skills (Track 2 entry)
- Becomes the public good for the ecosystem post-hackathon

This is **lower-effort, lower-ceiling, but higher-floor**. Even a bad PnL week leaves a useful primitive. And it positions the operator as "the indexer for agent reputation on BNB" — a sticky post-hackathon role.
